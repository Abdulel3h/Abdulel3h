# Recruiter Review Path

This guide is for recruiters, hiring managers, and technical reviewers who want to understand the portfolio quickly without opening every repository.

## 30 Seconds

Confirm the core signal:

- Abdulelah is positioning for AI engineering, LLM applications, Cloud AI, and applied machine learning.
- The strongest public evidence is concentrated in six repositories, not spread evenly across every repo.
- Public claims are conservative: prototype boundaries, missing evidence, and future work are documented instead of hidden.

Start with:

1. [GitHub profile README](../README.md)
2. [Abdulelah AI Portfolio](https://github.com/Abdulel3h/Abdulelah)
3. [Live website](https://www.abdulelah.de)

## 5 Minutes

Open one README and one documentation file from each flagship repository:

- `README.md` for scope, setup, screenshots, limitations, and future work.
- `docs/architecture.md` for system flow and production gaps.
- `docs/reviewer-guide.md` for the fastest technical inspection path.
- `docs/technical-decisions.md` for design tradeoffs.

## Portfolio Interpretation

- Repositories marked as archive candidates are intentionally not part of the recruiter path.
- Learning projects remain public only as history; they are not presented as current AI engineering evidence.
- Internal or employer-owned work is not used as public portfolio evidence.
- Missing screenshots, metrics, or demos should be treated as known gaps unless a repository documents verified evidence.

## Best Interview Signals

- Ability to turn AI ideas into complete product surfaces.
- Careful separation between prototype work and production claims.
- Documentation that explains architecture, limitations, and next steps.
- Arabic-first product thinking across assistants, analytics, and advisory concepts.
- Practical use of Python, TypeScript, FastAPI, Flask, Next.js, TanStack, Ollama, and cloud-oriented deployment patterns.
